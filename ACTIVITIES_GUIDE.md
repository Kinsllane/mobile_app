# 📱 Руководство по взаимодействию активити (экранов)

## Обзор архитектуры

Приложение содержит **7 взаимосвязанных экранов (активити)**, которые активно взаимодействуют друг с другом через:
- **Навигацию** - переход между экранами с передачей данных
- **Общий контекст (Context API)** - единое хранилище данных для всех экранов
- **AsyncStorage** - постоянное хранилище данных

---

## 📋 Детальное описание экранов и их взаимодействия

### 1. **BooksListScreen** (Главный экран)
**Роль:** Центральный хаб приложения

**Взаимодействие с другими экранами:**
- ➡️ **StatisticsScreen** - кнопка "📊 Статистика" 
- ➡️ **GoalsScreen** - кнопка "🎯 Цели"
- ➡️ **FavoritesScreen** - кнопка "❤️ Избранное"
- ➡️ **BookDetailScreen** - клик на карточку книги (передаёт `bookId`)
- ➡️ **AddBookScreen** - кнопка "+ Добавить книгу"

**Получает данные из:**
- `BooksContext` - список всех книг
- `GENRES` - список доступных жанров

**Функции:**
- Отображение списка книг с фильтрацией и сортировкой
- Поиск по названию и автору
- Быстрая навигация к другим экранам

---

### 2. **StatisticsScreen** (Детальная статистика)
**Роль:** Аналитика и визуализация данных

**Взаимодействие:**
- ➡️ **GoalsScreen** - кнопка "🎯 Перейти к целям чтения"
- ⬅️ Получает данные о всех книгах из `BooksContext`

**Что показывает:**
- 4 карточки основных метрик (всего книг, прочитано, средний рейтинг, прогресс)
- 2 дополнительные карточки (высокий рейтинг, читаю сейчас)
- Прогресс-бары по статусам (прочитано, читаю, планирую, отложено)
- Топ-5 популярных жанров с визуализацией

**Передаёт в GoalsScreen:**
- Навигационный переход для продолжения работы с данными

---

### 3. **GoalsScreen** (Управление целями)
**Роль:** Постановка и отслеживание целей чтения

**Взаимодействие:**
- ➡️ **StatisticsScreen** - кнопка "📊 Статистика"
- ➡️ **BooksListScreen** - кнопка "📚 К списку книг"
- ⬅️ Получает данные о книгах из `BooksContext`
- 💾 Сохраняет цели в `AsyncStorage` (ключ: `@reading_goals`)

**Что хранит:**
```javascript
{
  yearlyGoal: число,   // Годовая цель
  monthlyGoal: число   // Месячная цель
}
```

**Логика взаимодействия:**
- Анализирует книги по датам добавления (из `book.id`)
- Подсчитывает прогресс за текущий год и месяц
- Сравнивает с установленными целями
- Показывает процент выполнения через компонент `ProgressBar`

---

### 4. **FavoritesScreen** (Избранное)
**Роль:** Отображение избранных книг

**Взаимодействие:**
- ➡️ **BookDetailScreen** - клик на карточку книги (передаёт `bookId`)
- ➡️ **BooksListScreen** - кнопка "📚 К списку книг" (если пусто)
- ⬅️ Получает книги с флагом `isFavorite: true` из `BooksContext`

**Логика фильтрации:**
```javascript
const favoriteBooks = books.filter((book) => book.isFavorite);
```

---

### 5. **BookDetailScreen** (Детали книги)
**Роль:** Полная информация о книге и управление

**Взаимодействие:**
- ➡️ **EditBookScreen** - кнопка "Редактировать" (передаёт `bookId`)
- ⬅️ Получает данные из `BooksContext.getBookById(bookId)`
- 💾 Обновляет книгу через `updateBook(bookId, data)`
- 🗑️ Удаляет книгу через `deleteBook(bookId)` → возврат назад

**Изменяемые данные:**
- `isFavorite` - переключение избранного (влияет на FavoritesScreen)
- `notes` - заметки о книге
- `keyPoints` - ключевые моменты (массив строк)

**Модальные окна:**
- Редактирование заметок
- Редактирование ключевых моментов

---

### 6. **AddBookScreen** (Добавление книги)
**Роль:** Создание новой книги

**Взаимодействие:**
- ⬅️ Открывается из `BooksListScreen`
- 💾 Создаёт книгу через `BooksContext.addBook(bookData)`
- ⬅️ После сохранения → возврат на главный экран

**Создаваемые данные:**
```javascript
{
  id: timestamp,
  title: строка,
  author: строка,
  rating: число (0-5),
  status: строка,
  genre: строка,
  summary: строка,
  coverColor: случайный цвет,
  isFavorite: false,
  notes: '',
  keyPoints: []
}
```

---

### 7. **EditBookScreen** (Редактирование книги)
**Роль:** Изменение данных существующей книги

**Взаимодействие:**
- ⬅️ Открывается из `BookDetailScreen` (получает `bookId`)
- 💾 Обновляет через `BooksContext.updateBook(bookId, newData)`
- ⬅️ После сохранения → возврат на экран деталей

---

## 🔗 Компоненты взаимодействия

### Context API (BooksContext)
Центральное хранилище данных, используемое ВСЕМИ экранами:

```javascript
{
  books: [],              // Массив всех книг
  loading: boolean,       // Состояние загрузки
  addBook: function,      // Добавить книгу
  updateBook: function,   // Обновить книгу
  deleteBook: function,   // Удалить книгу
  getBookById: function   // Получить книгу по ID
}
```

### AsyncStorage
Постоянное хранилище:
- `@books_storage` - все книги
- `@reading_goals` - цели чтения

### Навигация (React Navigation)
Передача данных между экранами:
```javascript
// Передача
navigation.navigate('BookDetail', { bookId: '123' });

// Получение
const { bookId } = route.params;
```

## 📊 Примеры взаимодействия

### Пример 1: Добавление в избранное
```
1. Пользователь на BooksListScreen
2. Нажимает на книгу → BookDetailScreen
3. Нажимает кнопку "🤍 Добавить в избранное"
4. BookDetailScreen вызывает updateBook(bookId, {isFavorite: true})
5. BooksContext обновляет данные в AsyncStorage
6. Пользователь переходит в FavoritesScreen
7. FavoritesScreen фильтрует и показывает книги с isFavorite: true
```

### Пример 2: Постановка цели и отслеживание прогресса
```
1. Пользователь на BooksListScreen
2. Нажимает кнопку "🎯 Цели"  → GoalsScreen
3. Устанавливает годовую цель: 12 книг
4. GoalsScreen сохраняет в AsyncStorage
5. GoalsScreen анализирует books из BooksContext
6. Подсчитывает прочитанные книги за год
7. Показывает прогресс: 5/12 книг (42%)
8. Пользователь нажимает "📊 Статистика" → StatisticsScreen
9. StatisticsScreen показывает детальную визуализацию тех же данных
```

### Пример 3: Редактирование книги
```
1. FavoritesScreen → клик на книгу → BookDetailScreen
2. BookDetailScreen → кнопка "Редактировать" → EditBookScreen
3. EditBookScreen изменяет данные → вызывает updateBook()
4. BooksContext обновляет данные
5. Возврат на BookDetailScreen → данные обновлены
6. Все экраны (BooksListScreen, StatisticsScreen, FavoritesScreen) 
   автоматически видят изменения через Context
```

## 🎓 Заключение

Приложение демонстрирует:
- ✅ **7 взаимосвязанных активити/экранов**
- ✅ **Двунаправленное взаимодействие** (туда-обратно)
- ✅ **Общее состояние** через Context API
- ✅ **Постоянное хранилище** через AsyncStorage
- ✅ **Передачу данных** через параметры навигации
- ✅ **Реактивные обновления** - изменения в одном экране видны в других

Все экраны работают как единая система, обмениваясь данными и функциями.
