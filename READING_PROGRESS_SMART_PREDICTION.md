# Умное предсказание завершения чтения ✅

## Проблема (до)

Была ошибка в логике расчёта прогноза:
```javascript
const pagesPerDay = currentPage / daysReading; // ❌ Неправильно!
```

**Пример:**
- День 1: Пользователь прочитал 16 страниц → `startDate` = сегодня
- День 2: Прочитал ещё 5 страниц, обновил `currentPage = 21`
- Прогноз считал: 21 страницу / 1 день = **21 стр/день** 😱

**Результат:** Прогноз был неточным и не учитывал реальный темп чтения.

---

## Решение (после)

### 1. История чтения `readingHistory`

Теперь каждое изменение `currentPage` **автоматически сохраняется** в массиве:

```javascript
readingHistory: [
  { date: "2026-06-20T10:00:00.000Z", pages: 16 },
  { date: "2026-06-21T14:30:00.000Z", pages: 21 },
  { date: "2026-06-22T19:00:00.000Z", pages: 35 },
]
```

### 2. Автоматическое обновление истории

В `BooksContext.js → updateBook()`:

```javascript
// При изменении currentPage - добавляем запись в историю
if (updatedData.currentPage !== undefined && updatedData.currentPage !== book.currentPage) {
  const today = new Date().toISOString().split('T')[0];
  
  // Если запись за сегодня уже есть - обновляем её
  // Если нет - добавляем новую
  currentHistory.push({
    date: new Date().toISOString(),
    pages: updatedData.currentPage,
  });
}
```

### 3. Умный расчёт прогноза

В `ReadingProgress.js → getEstimatedFinishDate()`:

**Приоритет 1: История чтения** (если есть >= 2 записей)
```javascript
// Берём последние 7 записей
const recentHistory = readingHistory.slice(-7);

// Считаем средний темп между записями
for (let i = 1; i < recentHistory.length; i++) {
  const pagesRead = currEntry.pages - prevEntry.pages; // Разница страниц
  const daysDiff = (currDate - prevDate) / (1000 * 60 * 60 * 24); // Разница дней
  
  totalPagesRead += pagesRead;
  totalDays += daysDiff;
}

const pagesPerDay = totalPagesRead / totalDays; // ✅ Правильно!
```

**Приоритет 2: Простой расчёт** (fallback если истории нет)
```javascript
const daysReading = (now - start) / (1000 * 60 * 60 * 24);
const pagesPerDay = currentPage / daysReading;
```

## Как это работает для пользователя

### Сценарий: Чтение книги 300 страниц

| День | Действие | currentPage | История | Прогноз темпа |
|------|----------|-------------|---------|---------------|
| **1** | Начал читать, 16 страниц | 16 | `[{date: день1, pages: 16}]` | 16 стр/день |
| **2** | +5 страниц, меняет на 21 | 21 | `[{день1: 16}, {день2: 21}]` | (21-16)/1 = **5 стр/день** ✅ |
| **3** | +14 страниц, меняет на 35 | 35 | `[{день1: 16}, {день2: 21}, {день3: 35}]` | ((21-16) + (35-21))/2 = **9.5 стр/день** |
| **7** | Прочитал много | 120 | 7 записей | Средний темп за 7 дней |

### Преимущества:

✅ **Точный прогноз**: Учитывается реальный темп чтения  
✅ **Адаптивный**: Обновляется каждый раз при изменении страницы  
✅ **Умный**: Берёт последние 7 записей (не весь период)  
✅ **Защита от выбросов**: Если пользователь читал медленно в начале, а потом ускорился - прогноз адаптируется

## Технические детали

### Обновление одной записи в день

Если пользователь несколько раз за день меняет `currentPage`:
```javascript
const today = new Date().toISOString().split('T')[0]; // "2026-06-24"

// Проверяем, есть ли запись за сегодня
const todayIndex = currentHistory.findIndex(entry => 
  entry.date.split('T')[0] === today
);

if (todayIndex >= 0) {
  // Обновляем запись за сегодня (не дублируем)
  currentHistory[todayIndex] = { date: new Date().toISOString(), pages: 21 };
}
```

### Fallback для старых книг

Если книга была добавлена до обновления (нет `readingHistory`):
- Используется старый метод расчета от `startDate`
- При первом обновлении `currentPage` - история начнёт создаваться автоматически

## Примеры расчётов

### Пример 1: Равномерное чтение
```
История: [
  { день1: 20 стр },
  { день2: 40 стр },
  { день3: 60 стр },
]

Темп: ((40-20) + (60-40)) / 2 = 20 стр/день
Осталось: 300 - 60 = 240 стр
Прогноз: 240 / 20 = 12 дней
```

### Пример 2: Ускорение
```
История: [
  { день1: 10 стр },   // медленно начал
  { день5: 30 стр },   // 5 стр/день
  { день7: 70 стр },   // 20 стр/день (ускорился!)
]

Темп последних записей: ((30-10)/4 + (70-30)/2) / 2 = 12.5 стр/день
(берёт последние записи, где темп выше)
```

### Пример 3: Торможение
```
История: [
  { день1: 50 стр },   // быстро начал
  { день2: 80 стр },   // 30 стр/день
  { день5: 95 стр },   // 5 стр/день (замедлился)
]

Темп: адаптируется к последним данным
```
